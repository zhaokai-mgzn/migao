# AI 验收官使用手册（凯总/娜总）

## 1. 怎么提一个可被自动验收的 issue

### 步骤 1：选模板
- 进入 "Issue" → "New issue"
- 选 `功能验收 (AI 验收)` 模板
- 模板已强制 "业务真值" 段

### 步骤 2：写业务真值（关键）
用业务语言，**不带技术性**：

✅ 好的例子：
- 含加工待发货 = 状态为待发货 且 含加工项
- 看板跳转后筛选条件 = 卡片标题一致
- 客户搜索支持手机号/姓名

❌ 不好的例子：
- SELECT COUNT(*) FROM orders WHERE status='pending_shipment' AND has_processing=true
- HTTP GET /api/dashboard/pending-shipment-count

### 步骤 3：等军师反推 case
- 提交 issue 后 1-5 分钟内，军师会自动评论 case 草稿
- 草稿包含 L2（单测）+ L3（E2E）+ L4（业务断言）

### 步骤 4：review 草稿
- ✅ 同意 → 不用动，研发认领后会按草稿写
- ❌ 不同意 → 评论 "X case 不合理，原因是 [Y]"

## 2. 怎么看待办报告

每天 19:00 群里会收到：

```
📋 今日验收报告
✅ 通过: 3 个 (#387 #390 #382)
❌ 失败: 1 个 (#XXX，跳转URL错)
⏳ 待验收: 5 个
```

**通过的不用管**。**失败的看群里附的详细报告**（commit、case、原因）。

## 3. 凯总/娜总需要做什么

| 阶段 | 动作 | 时间 |
|---|---|---|
| 提 issue | 写业务真值 | 5-10 min |
| 验收 | 看报告 | 1 min |
| 抽样复核 | 每周抽 1-2 个 | 5 min |

**总投入：~10 min/issue**（之前 90 min）。

## 4. 异常处理

| 情况 | 怎么做 |
|---|---|
| 军师反推 case 错了 | 评论 "X case 不合理" + 原因，军师会改模板 |
| 验收失败但认为是误报 | 评论 "误报，case 描述 X 实际是 Y" |
| 凯总/娜总推翻 AI 决定 | 加 `override/keep-open` label + 评论原因 |
| 红牌有 bug | 验收会自动发现，群里会告警 |
