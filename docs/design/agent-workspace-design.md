# 客服员工工作台产品设计

> 版本：v8.0  
> 日期：2026-04-12  
> 范围：客服员工小程序 + PC H5 工作台 / 会话分配引擎 / 转人工流程

---

## 一、产品定位

客服员工工作台是企业管理员和客服员工处理"转人工"会话的核心工具。员工无需下载额外应用，通过微信小程序即可随时处理客户咨询，PC 端也可通过浏览器访问 H5 版本。

**核心用户**：
- **客服主管**：管理团队、查看数据、处理复杂会话
- **普通客服**：接收分配会话、回复客户、处理工单

**接入方式**：
| 端 | 入口 | 登录方式 | 适用场景 |
|------|------|---------|---------|
| 移动端（主） | 微信"客服工作台"小程序 | 微信授权 + 手机号验证 | 随时随地处理会话 |
| PC 端（辅） | 浏览器访问 H5 工作台 | 微信扫码登录 | 多会话并行处理 |

## 三、核心流程

### 3.1 员工首次使用流程

```
管理员在管理后台添加客服员工
    │
    ├─ 输入：姓名、手机号、角色（客服主管/普通客服）
    │
    ├─ 系统生成账号 → 创建"邀请二维码"和"邀请链接"
    │
    ▼
员工在微信中扫描二维码（或点击链接）
    │
    ├─ 进入"客服工作台"小程序
    │
    ├─ 微信授权登录 → 获取 openid + unionid
    │
    ├─ 验证手机号（短信验证码，与管理员录入的手机号匹配）
    │
    ├─ 设置昵称和头像（可选，默认用微信信息）
    │
    ├─ 自动上线 → 进入工作台主页
    │
    └─ 提示"添加到我的小程序" → 以后微信下拉即可进入
```

### 3.2 转人工会话分配流程

```
用户在 AI 对话中触发"转人工"
    │
    ├─ ai-agent-service 检测到转人工关键词或用户主动点击
    │
    ├─ AI 发送结束语："正在为您转接人工客服，请稍候..."
    │
    ▼
ai-agent-service 调用 admin-api 创建人工服务工单
    │
    ├─ POST /api/internal/handoff
    │    { sessionId, reason, priority }
    │
    ▼
admin-api 会话分配引擎执行
    │
    ├─ 查询当前在线的客服员工
    │
    ├─ 如果在线员工 > 0：
    │   └─ 按轮询策略分配给当前接待量最少的员工
    │       ├─ 员工收到小程序订阅消息："新会话，请处理"
    │       └─ 员工点击消息 → 进入小程序 → 看到会话
    │
    └─ 如果在线员工 = 0：
        └─ 进入排队队列 → 用户收到"当前无在线客服，请留言"
            ├─ 队列中的会话按优先级排序
            └─ 员工上线后自动分配队列中最优先的会话
```

### 3.3 员工处理会话流程

```
员工在小程序中打开新会话
    │
    ├─ 查看客户信息卡片
    │   ├─ 基本信息（昵称、头像、注册时间）
    │   ├─ 历史订单（最近 3 笔）
    │   └─ 对话历史（AI 对话记录，了解上下文）
    │
    ├─ 回复客户
    │   ├─ 手动输入文字
    │   ├─ 选择快捷回复模板
    │   └─ 发送图片/文件（可选）
    │
    ├─ 会话操作
    │   ├─ 转接给其他员工
    │   ├─ 添加内部备注（客户不可见）
    │   └─ 打标签（咨询/投诉/售后/其他）
    │
    └─ 结束会话
        ├─ 点击"结束会话"
        ├─ 邀请用户评价（可选）
        └─ 会话归档 → 状态变为"已结束"
```

### 3.4 员工上下线管理

```
员工进入小程序 → 自动上线
    │
    ├─ 状态变为"在线" → 开始接收新会话
    │
    ├─ 员工点击"小休" → 状态变为"忙碌"
    │   └─ 不再分配新会话，但当前会话继续
    │
    ├─ 员工点击"下班" → 状态变为"离线"
    │   └─ 不再分配新会话
    │   └─ 当前未结束的会话自动转接给其他在线员工
    │
    └─ 管理员可在后台强制上下线员工
```

### 9.2 agent_sessions（人工会话表）

```sql
CREATE TABLE agent_sessions (
    id VARCHAR(64) PRIMARY KEY,
    tenant_id VARCHAR(64) NOT NULL REFERENCES tenants(id),
    user_id VARCHAR(64) NOT NULL REFERENCES users(id),  -- 客户 ID
    employee_id VARCHAR(64) REFERENCES agent_employees(id),  -- 分配给的员工
    ai_session_id VARCHAR(64),  -- 关联的 AI 会话 ID
    status VARCHAR(32) DEFAULT 'waiting',  -- waiting / active / queued / ended
    priority INTEGER DEFAULT 0,  -- 优先级（投诉=1, 普通=0）
    reason TEXT,  -- 转人工原因
    queue_position INTEGER,  -- 排队位置
    wait_time_seconds INTEGER DEFAULT 0,  -- 等待时长
    response_time_seconds INTEGER,  -- 首次响应时长
    total_messages INTEGER DEFAULT 0,  -- 总消息数
    tags JSONB DEFAULT '[]',  -- 会话标签
    rating INTEGER,  -- 用户评分 1-5
    rating_comment TEXT,  -- 用户评价
    ended_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE INDEX idx_agent_sessions_employee ON agent_sessions(employee_id, status);
CREATE INDEX idx_agent_sessions_user ON agent_sessions(user_id);
CREATE INDEX idx_agent_sessions_queued ON agent_sessions(status, priority, created_at)
    WHERE status = 'queued';
```

## 十一、AI 转人工桥接流程

### 11.1 切换机制

```
AI 对话中 → 用户触发转人工
    │
    ├─ ai-agent-service 执行以下步骤：
    │
    ├─ 1. 停止 AI 自动回复
    │
    ├─ 2. 将当前 session 标记为 "handoff_requested"
    │
    ├─ 3. 调用 admin-api 创建 agent_session
    │   POST /api/admin/agent/tickets
    │   {
    │     user_id: "user_xxx",
    │     ai_session_id: "sess_xxx",
    │     reason: "用户请求转人工",
    │     priority: 0
    │   }
    │
    ├─ 4. admin-api 返回 agent_session_id
    │
    ├─ 5. ai-agent-service 向用户发送转接中消息
    │   "正在为您转接人工客服，请稍候..."
    │
    └─ 6. 用户端切换到人工会话 WebSocket 通道
        （小程序端：切换到 agent-session 页面）
        （H5 端：WebSocket 切换到 agent channel）
```

### 11.2 消息通道切换

```
用户端消息流:

AI 阶段:
用户消息 → API Gateway → ai-agent-service → Hermes Agent → AI 回复 → SSE 返回用户

转人工后:
用户消息 → API Gateway → admin-api → WebSocket → 客服员工小程序/H5
                                      ↓
                              客服员工回复 → WebSocket → admin-api → SSE → 用户
```

### 12.2 WebSocket 实时对话

```javascript
// 小程序端 WebSocket 连接
const ws = wx.connectSocket({
  url: `wss://api.migaozn.com/ws/agent?token=${token}`
});

// 监听消息
ws.onMessage((res) => {
  const event = JSON.parse(res.data);
  switch (event.event) {
    case 'new_session':
      // 新会话通知，更新列表
      break;
    case 'session_message':
      // 收到客户消息，更新对话区
      break;
    case 'queue_alert':
      // 排队告警，显示提示
      break;
  }
});

// 发送消息
function sendMessage(sessionId, content) {
  ws.send({
    data: JSON.stringify({
      event: 'send_message',
      data: { session_id: sessionId, content, content_type: 'text' }
    })
  });
}
```
