# 企业管理后台产品设计

> 版本：v8.0
> 日期：2026-04-12
> 定位：单租户商家后台（多租户技术架构，企业管理员视角）
> 技术栈：Next.js SSR 部署到 SAE

---

## 一、产品定位与架构

### 1.1 产品定位

企业管理后台是 AI 客服系统的**运营管理中枢**，面向企业内部不同角色的员工，提供：

- **混合数据看板**：业务指标（订单/营收）+ 客服指标（会话量/转人工率/满意度）
- **AI 客服配置**：自定义 AI 自我介绍、营业时间、转人工规则、快捷回复
- **客服团队管理**：员工管理、会话分配监控、服务质量统计
- **业务管理**：商品、订单、售后工单
- **RBAC 权限**：按角色控制菜单可见性和数据操作权限

### 1.2 多租户与单租户视角

```
技术架构层（多租户）                    商家后台层（单租户视角）
┌─────────────────────────────┐       ┌──────────────────────────┐
│  平台运营后台（独立系统）     │       │  企业管理后台（当前产品）  │
│  - 租户列表与计费            │       │  - 只看本租户数据          │
│  - 用量统计与配额管理         │       │  - JWT 中强制携带 tenant_id│
│  - 平台级监控               │       │  - 菜单按员工角色动态显示   │
└─────────────────────────────┘       └──────────────────────────┘
```

**关键设计原则**：
- 商家后台不出现"租户管理"菜单（由平台运营侧管理）
- 所有 API 请求的 `tenant_id` 从 JWT Claims 中提取，禁止客户端传递
- 数据库所有业务表含 `tenant_id` 字段，PostgreSQL RLS 强制隔离

### 1.3 角色与权限矩阵

| 角色 | 代码 | 菜单权限 | 数据权限 |
|------|------|---------|---------|
| **超级管理员** | super_admin | 全部菜单 | 本租户全部数据 |
| **运营经理** | operation_manager | 数据看板、AI 配置、客服团队、业务管理 | 本租户全部数据 |
| **客服主管** | support_supervisor | 数据看板、客服团队、快捷回复 | 本租户客服数据 |
| **客服员工** | support_agent | 数据看板（仅客服指标）、我的会话、快捷回复 | 仅本人处理的会话 |
| **商品管理员** | product_manager | 商品管理、订单管理（只读） | 本租户商品/订单数据 |

**RBAC 实现方式**：
- 前端：根据用户角色动态渲染菜单，无权限菜单项隐藏
- 后端：每个 API 校验 JWT 中的 `role` Claim，越权返回 403
- 数据层：PostgreSQL RLS 策略，即使后端绕过校验也无法跨租户/跨角色访问

#### 6.5.3 工单处理流程

**退款工单处理流程**：
1. 客户提交退款申请（C 端订单详情页自助申请，或客服代创建）
2. 系统自动分配（按轮询规则分配给在线客服）
3. 客服审核：查看订单信息、客户凭证、聊天记录
4. 客服处理：同意退款 / 拒绝退款（需填写原因）/ 要求补充凭证
5. 客户确认：同意退款方案 → 系统执行退款；拒绝 → 返回客服重新处理
6. 退款执行：调用微信支付退款 API，原路退回
7. 工单完成：记录处理结果，更新订单状态

**换货工单处理流程**：
1. 客户提交换货申请
2. 客服审核：确认换货原因、库存情况
3. 客服处理：确认换货方案（同色/异色/不同规格）
4. 客户寄回：生成退货物流单，客户寄回商品
5. 仓库收货：确认商品状态，发起换货出库
6. 新商品发货：生成新物流单，通知客户
7. 工单完成

**补发工单处理流程**：
1. 客户提交补发申请（商品缺失/损坏）
2. 客服审核：查看客户提供的凭证图片
3. 客服处理：确认补发商品、数量
4. 仓库出库：生成补发出库单
5. 补发发货：生成物流单，通知客户
6. 工单完成

### 9.3 SSR 认证中间件

```typescript
// middleware/auth.ts
import { NextRequest, NextResponse } from 'next/server';
import { verifyJwt } from '@/lib/jwt';

export async function middleware(request: NextRequest) {
  const token = request.cookies.get('access_token')?.value;

  if (!token) {
    return NextResponse.redirect(new URL('/login', request.url));
  }

  try {
    const payload = await verifyJwt(token);

    // 验证 tenant_id 存在
    if (!payload.tenant_id) {
      return NextResponse.redirect(new URL('/login', request.url));
    }

    // 将用户信息注入请求头（供 API 路由使用）
    const requestHeaders = new Headers(request.headers);
    requestHeaders.set('x-user-id', payload.user_id);
    requestHeaders.set('x-tenant-id', payload.tenant_id);
    requestHeaders.set('x-user-role', payload.role);

    return NextResponse.next({
      request: { headers: requestHeaders }
    });
  } catch {
    return NextResponse.redirect(new URL('/login', request.url));
  }
}

export const config = {
  matcher: ['/((?!login|_next/static|_next/image|favicon.ico).*)']
};
```

### 9.4 WebSocket 实时数据

客服状态监控、会话监控等需要实时数据的页面使用 WebSocket 连接：

```typescript
// hooks/useWebSocket.ts
import { useEffect, useState, useCallback } from 'react';

export function useAgentWebSocket(token: string) {
  const [ws, setWs] = useState<WebSocket | null>(null);
  const [sessions, setSessions] = useState([]);
  const [agents, setAgents] = useState([]);

  useEffect(() => {
    const socket = new WebSocket(`wss://api.migaozn.com/ws/agent?token=${token}`);

    socket.onmessage = (event) => {
      const { event: type, data } = JSON.parse(event.data);

      switch (type) {
        case 'session_update':
          setSessions(prev => updateSession(prev, data));
          break;
        case 'agent_status_change':
          setAgents(prev => updateAgent(prev, data));
          break;
        case 'new_session':
          setSessions(prev => [...prev, data]);
          break;
      }
    };

    setWs(socket);
    return () => socket.close();
  }, [token]);

  return { sessions, agents };
}
```
