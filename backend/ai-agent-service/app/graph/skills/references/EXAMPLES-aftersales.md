# After-Sales Skill — Few-shot 示例

## ✅ 正确示例

### 例1: 创建售后工单
用户: "ORD-1234 的客户要退款，说不合适"
```
→ 先查订单: order_query(action=detail, order_no="ORD-1234")
→ 了解订单详情后确认问题
→ ask: "请确认退款原因和金额？订单金额¥264，客户说'不合适'"
→ 用户: "全退，金额264"
→ confirm: "确认创建退款工单？ORD-1234 张三 ¥264 原因:不合适"
→ after_sales_manage(action=create, ...) → "工单已创建 AS-xxx"
```

### 例2: 查看工单进度
用户: "上周那个退款工单处理完了吗？"
```
→ after_sales_manage(action=list, status="pending")
→ 📋 待处理工单
  AS-xxx · ORD-1234 · 退款 · 申请中 · 06-05
→ 工单还在处理中，需要催一下吗？
```

### 例3: 修改售后方案
用户: "换个方案，只退一半"
```
→ 识别修正意图 → 更新 reason/amount
→ "好的，已更新退款方案：退款¥132（50%）。确认创建工单？"
```

## ❌ 错误示例

### 反例1: 未查订单直接处理
用户: "ORD-1234 退货"
```
❌ "好的，已创建退货工单"  （没查订单详情）
✅ 先查 order_query → 确认订单存在 + 了解详情 → 再处理
```

### 反例2: 编造售后政策
用户: "退货要收手续费吗？"
```
❌ "退货需要收10%手续费"  （未核实公司政策）
✅ "关于退货手续费，这属于具体售后政策，建议查看公司售后规则或咨询主管"
```

### 反例3: 复杂投诉不转人工
用户: "你们商品有质量问题，我要求三倍赔偿"
```
❌ "好的，我来帮您处理赔偿"  （涉及法律/赔偿应转人工）
✅ "这涉及赔偿问题，需要人工客服介入处理。我会帮您记录情况并转交专人跟进。"
```
